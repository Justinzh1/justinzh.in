Hermione

Full Stack App built with the MEAN stack.

Hermione is a computer vision project that was created within the ASUC CTO division. It was created as a means to help professors record and post their lectures in an easily accessible manner. The site supports user authentication using a MongoDB database to allow professors to create new classes, post new videos, and manage students. Students on the otherhand have the ability to enroll in classes, and view classes. Hermione serves as a clean portal for students and professors record and review lectures at any time.


The front end is compiled using jade (now renamed to pug) to modularize components of web page. To handle user authentication I used the passport module in conjunction with a MongoDB database to serialize and hash user passwords to ensure security. Passport allows for authentication via Google accounts, which is important for ensuring that users are actually enrolled in the class and attend UC Berkeley. To connect to the database, I used mongoose to create respective collections to create class objects for storing links to the recorded lectures.

CPU

Buildling a pipelined CPU in Logisim

In our largest project in my Computer Architecture class I was tasked with creating a fully functional CPU to support basic MIPS operations (R-type, I-type and J-type instructions) from scratch.


First I began by creating the regfile. Since I wanted to create a 32 bit machine, I created 32 registers in the regfile module, making sure to link the clock to all of them and added three inputs on the left most side (rs, rt, rw), two switches on the bottom and 2 output links on the right side so the values could be processed by the ALU later.


Next, I set out to create a functional ALU to handle the basic ALU operations that include sll, srl, sra, mult, divu, add, addu, sub, subu, and, or, xor, nor, slt and sltu. In my design process I decided that the optimal solution would be to calculate the result for every possible operation since they run in parallel and then use a MUX to determine which final result to output. My ALU took in 3 values (x, y, and switch) and outputed 4 values (overflow, equals, result_lo, and result_hi).


After implementing the ALU, I created the controller for the CPU, this functioned as a series of comparators to determine the opcode, func and immediate for each inputed command. Depending on the command it would activate specific indicators used to mark branching, sign extending, write to memory, etc. These commands were used in conjunction with MUX's to have the expected behavior of a normal CPU.


Additionally I used splitters to decode the instruction and a register directly before the decode process to create a simple 2-stage pipeline. To handle jumps and branches, I calculated the possible different values that the new PC could be after processing the command and used a MUX from the controller I made earlier to decide which value to use. Writing back to memory and registers was handled in a similar manner as well.

Cyrcle

Front End Developement working with Algolia.js

Over the summer I teamed up with Berkeley club, CodeBase to work with a local San Francisco startup called Cyrcle. Cyrcle is an e-commerce startup with a focused on helping local small businesses get their products on the market as quickly and as easily as possible.


As the front end engineer, I set out to create a minimal product that emphasized the search component to highlight the fast speeds that Algolia.js provides. In addition to designing and building the entire front end, I was also responsible for creating filters to refine search results so users would get the information that they actually want.
